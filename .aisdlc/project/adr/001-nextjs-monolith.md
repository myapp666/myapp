# ADR-001：Next.js 单体应用架构

## 决策

采用 **Next.js 15 单体应用** 作为竞争情报监控系统的主体架构。

## 理由

### 优势
1. **快速迭代**：前后端一体，无需跨仓库同步；API Routes 清晰边界
2. **部署简单**：Vercel 原生支持，自动 CI/CD，零配置 Cron
3. **开发体验**：TypeScript、HMR、Tailwind 集成，生态成熟

### 约束
- MVP 阶段优先快速迭代而非微服务隔离
- 采集/AI 功能内聚在单体，后期可独立为 microservice

## 权衡

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Next.js 单体** | 快速部署、开发体验好、Vercel 亲和 | 单点失败风险、后期扩展需重构 |
| 微服务 | 模块解耦、独立扩展 | MVP 阶段过度设计、部署复杂 |
| API-first 分离 | 前后端独立迭代 | 沟通成本高、网络开销 |

## 后续演进

若后期需扩展采集/AI 服务，可：
1. 独立为 Node.js microservice（与单体通过 API 调用）
2. 使用 Job Queue（Bull/RabbitMQ）解耦采集任务
3. 保持 Vercel 部署单体前端 + 其他平台部署服务
